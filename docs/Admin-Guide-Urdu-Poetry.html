Urdu Poetry — Admin Guide

What administrators can do in the system · Generated June 2026

How someone becomes admin

Admin access is checked when either:

  • The user's profile has user_role = 'admin' in Supabase, or
  • They log in as admin@urdupoetry.com (hardcoded fallback for local/demo use)

To set the first real admin in Supabase, run supabase/migrate-user-role.sql in the Supabase SQL Editor (it can promote a user by email).

Where admins go

  • Route: #/admin (Admin Dashboard)
  • Link: Settings → Admin Panel (only shown if the user is an admin)

Non-admins see "Admin Access Required" on that page.

What admins can do (Admin Dashboard)

1. Manage user roles

  • See all registered users (email, name, username)
  • Change any user between user and admin
  • Changes are saved to Supabase via set_user_role
  • Rule: An admin cannot remove their own admin role

2. Manage writing tags

  • Add, edit, and delete the scrolling tags in the Write / compose window (Urdu + English labels)
  • Save Tags writes to Supabase writing_tags (admin-only in database)

3. Review reports (moderation)

  • See pending reports (posts or users reported by others)
  • Approve or Remove each report
  • Updates report status in Supabase (admin-only policies on reports)

4. Contest overview (read-only)

  • View active contests (title, entries, deadline, prize)
  • View Submissions links to contests page — no full contest create/edit/delete in admin yet

Backend powers (API / Supabase) — not all have UI

Capability In admin UI?
Set featured poem (featured_poem table) No UI — API only (API.setFeaturedPoem)
Update any user profile (database policy) No dedicated UI — policy allows admins to update profiles
View / update all reports Partial — UI uses local storage; Supabase sync when online

What admins cannot do (by design)

  • End someone else's live mushaira — only the event host can use End Event
  • Delete any poem/post globally — no admin "delete content" tool in the panel
  • Manage voice rooms for others — no admin voice-room controls
  • Ban users — only role change (user/admin), no ban or suspend flow

Admin vs regular user vs mushaira host

Summary: Admins run the Admin Dashboard — user roles, writing tags, and report moderation — plus backend access to featured poem and profile updates. They do not automatically control live mushairas or all site content unless they are also the host.
